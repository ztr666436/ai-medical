# ai-medical-care-agents 架构设计方案

> 智能健康管理 Agents 项目 — 基于 Spring Cloud 微服务 + AI Agent 的蚂蚁阿福风格健康管理平台

---

## 一、项目定位

将传统健康管理 App 的核心能力（数据记录、目标管理、健康建议）与 AI Agent 的对话式交互能力融合，打造一个**松耦合、可扩展、智能化**的 Web 端健康管理平台。风格上模仿蚂蚁阿福 App 的温馨体验——萌趣的卡通健康助手通过自然语言对话帮助用户完成健康管理。

### 2.2 各服务职责

| 服务 | 端口 | 核心职责 | 数据存储 |
|------|------|---------|---------|
| **gateway** | 8080 | 统一入口、路由转发、Token 校验、限流、CORS | 无状态 |
| **auth-service** | 8081 | 注册、登录、Token 签发/刷新、验证码 | MySQL (user_credential) + Redis (token) |
| **user-service** | 8082 | 用户信息管理、健康档案、偏好设置 | MySQL (user_profile) |
| **health-data-service** | 8083 | 健康数据 CRUD、对话式录入、数据统计 | MySQL (health_record) |
| **goal-service** | 8084 | 目标推荐、目标管理、进度追踪 | MySQL (health_goal + goal_progress) |
| **ai-agent-service** | 8085 | AI 对话、意图识别、实体抽取、健康建议生成 | 无 (调用外部 LLM API) |
| **common** | — | 通用实体、工具类、异常定义、统一响应 | 无 (被所有服务依赖) |

### 2.3 服务间通信

```
同步调用 (OpenFeign)：
  gateway ──feign──> auth-service (认证)
  ai-agent-service ──feign──> health-data-service (写入健康数据)
  ai-agent-service ──feign──> goal-service (查询/设置目标)

异步事件 (RabbitMQ)：
  health-data-service ──event──> goal-service (数据更新→目标进度检查)
  goal-service ──event──> ai-agent-service (目标达成→AI 生成祝贺消息)
```

## 五、核心数据库设计（概要）

### 5.1 认证相关

```sql
-- user_credential: 用户认证凭证
CREATE TABLE user_credential (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    phone VARCHAR(20) UNIQUE COMMENT '手机号（AES加密）',
    email VARCHAR(100) UNIQUE COMMENT '邮箱（AES加密）',
    password_hash VARCHAR(255) NOT NULL COMMENT 'BCrypt密码',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 5.2 健康数据

```sql
-- health_record: 健康数据记录
CREATE TABLE health_record (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT NOT NULL,
    record_type VARCHAR(20) COMMENT 'step/weight/blood_pressure/blood_sugar',
    record_value DECIMAL(10,2) NOT NULL,
    unit VARCHAR(10) COMMENT 'kg/mmHg/mmol/L',
    recorded_at DATETIME COMMENT '用户声称的记录时间',
    notes TEXT COMMENT '用户原始输入文本',
    confidence DECIMAL(3,2) COMMENT 'AI抽取置信度',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_user_date (user_id, recorded_at)
);
```

### 5.3 健康目标

```sql
-- health_goal: 健康目标
CREATE TABLE health_goal (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT NOT NULL,
    goal_type VARCHAR(20) COMMENT 'step/weight',
    target_value DECIMAL(10,2) NOT NULL,
    unit VARCHAR(10),
    period VARCHAR(10) COMMENT 'daily/weekly/monthly',
    status VARCHAR(10) DEFAULT 'active' COMMENT 'active/achieved/expired',
    ai_recommended BOOLEAN DEFAULT FALSE,
    started_at DATE,
    end_at DATE,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 六、接口规范 - 统一响应体

所有服务接口统一返回格式：

```java
// R.java
@Data
@NoArgsConstructor
@AllArgsConstructor
public class R<T> {
    private int code;      // 200=成功, 400=参数错误, 401=未认证, 500=系统异常
    private String message;
    private T data;

    public static <T> R<T> ok(T data) {
        return new R<>(200, "success", data);
    }
    public static <T> R<T> fail(int code, String message) {
        return new R<>(code, message, null);
    }
}
```

**响应示例：**

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "accessToken": "eyJhbGciOi...",
    "expiresIn": 7200
  }
}
```
