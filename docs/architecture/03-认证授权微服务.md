### 接口一览

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 注册 | POST | `/api/auth/register` | 手机号+密码+验证码 → Token |
| 登录 | POST | `/api/auth/login` | 手机号+密码 → Token |
| 刷新 | POST | `/api/auth/refresh` | Refresh Token → 新 Token 对 |
| 验证码 | POST | `/api/auth/send-code` | 发送短信验证码 |
| 健康 | GET | `/api/auth/health` | 服务健康检查 |

### 安全设计

| 层级 | 方案 | 位置 |
|------|------|------|
| 密码加密 | BCrypt | 入库时 `passwordEncoder.encode()` |
| 手机号/邮箱 | AES 加密存储 | 数据库字段（可逆加密） |
| Token 签发 | JWT (HS256) Access 2h + Refresh 7d | Common: JwtUtil |
| Token 黑名单 | Redis | 登出/刷新时写入 |
| 验证码 | Redis 5分钟过期 | `verify:code:{phone}` |
| 对外认证 | Gateway 统一 Token 校验 | auth-service 本身放行接口 |

### 关键流程

#### 注册流程
```
Client → POST /api/auth/register + body{phone, password, verifyCode}
  → AuthController.register()
  → AuthServiceImpl.register()
    1. Redis 校验验证码
    2. AES 加密手机号
    3. 检查是否已注册
    4. BCrypt 加密密码
    5. 入库 user_credential
    6. JwtUtil 生成 Access + Refresh Token
    7. Token 缓存到 Redis
  → 返回 R<TokenDTO>
```

#### 登录流程
```
Client → POST /api/auth/login + body{phone, password}
  → AuthServiceImpl.login()
    1. AES 加密手机号 → 查库
    2. BCrypt 匹配密码
    3. 签发 Token 对
    4. 更新 last_login_at
  → 返回 R<TokenDTO>
```

### 构建与运行

```bash
# 编译
mvn clean compile -pl ai-medical-care-auth

# 启动（确保 MySQL 和 Redis 已运行）
mvn spring-boot:run -pl ai-medical-care-auth
```

### 依赖方接入

其他微服务只需在 pom.xml 中引入 common 模块即可使用 `JwtUtil.validateToken()`：
```xml
<dependency>
    <groupId>com.medical</groupId>
    <artifactId>ai-medical-care-common</artifactId>
</dependency>
```
