### 6 个 REST 接口

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 对话录入 | POST | `/api/health/conversation` | 自然语言 → AI解析 → 入库 |
| 直接录入 | POST | `/api/health/record` | 结构化直接写入 |
| 今日数据 | GET | `/api/health/today` | 查询当日所有记录 |
| 按日查询 | GET | `/api/health/by-date?date=2026-05-26` | 指定日期 |
| 趋势数据 | GET | `/api/health/trend?type=step` | 近7天趋势（给图表） |
| 健康检查 | GET | `/api/health/health` | 服务状态 |

### 核心亮点

#### 1. 对话式录入流程
```
用户输入 "今天走了9000步，体重70kg"
  → HealthRecordController
  → HealthRecordServiceImpl.recordByConversation()
     → rules: AiAgentClient.parseHealthData()（Feign 调用 AI Agent）
     → fallback: parseWithRuleEngine()（正则兜底，无 AI 时自动降级）
  → 入库 health_record
  → eventPublisher.publishEvent(HealthDataUpdatedEvent)
     → goal-service 监听 → 检查目标进度
  → 返回 ConversationResponseDTO {
       reply: "收到！已为您记录：今日步数 9000 步...🚶 步数达标，太棒了！",
       records: [...],
       hasWarning: false
     }
```

#### 2. 规则引擎兜底（AI 不可用时自动降级）
- 步数：正则 `(\d+)\s*步`
- 体重：正则 `(\d+(?:\.\d+)?)\s*(kg|公斤|千克)`
- 血压：正则 `(\d+)\s*/\s*(\d+)`
- 血糖：正则 `(\d+(?:\.\d+)?)\s*(?:mmol/L|血糖)`
- 心率：正则 `(\d+)\s*(?:bpm|次/分钟|心率)`

#### 3. 事件驱动解耦
健康数据写入后发布 `HealthDataUpdatedEvent`，goal-service 异步监听处理目标进度检查，不阻塞主流程。

#### 4. AI Agent 集成预留
`AiAgentClient` 已定义 Feign 接口，待步骤6实现 ai-agent-service 后取消注释即可切换。
