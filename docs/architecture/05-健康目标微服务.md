## 关键创新：事件驱动自动进度检查

```
health-data-service 写入数据 → 发布 HealthDataUpdatedEvent
                                    ↓  @Async @EventListener
                          GoalServiceImpl.onHealthDataUpdated()
                                    ↓
                          检查同类型活跃目标
                          ├─ 达标 → 自动标记 achieved ✅
                          └─ 未达标 → 计算进度百分比 📊
```

## 5 个 REST 接口

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| AI推荐 | GET | `/api/goal/recommend?type=step` | 基于规则/AI推荐目标 |
| 创建/更新 | POST | `/api/goal` | 确认或修改推荐 |
| 活跃目标 | GET | `/api/goal` | 查询所有未完成目标 |
| 手动完成 | PUT | `/api/goal/{id}/achieve` | 标记目标达成 |
| 进度报告 | GET | `/api/goal/report` | 文字版进度汇总 |
